Specification: To insert a new word, starting from a blank tree.
  • Create a new empty tree and use a pointer from the head to address a new empty list which will be called: root list. It can be a general linear list.
  • For the first word: The tree is empty, and is as defined previously. And the word ACT is going to be inserted. As root list is empty, add a node to the root list with the char A. Then, create a new empty list, use a pointer from the node that is holding the A to point to this new list, create a node on this second list and take second letter C and place it on the node, create a new empty list, use a pointer from the node holding the C to point to that list. Do the same for letter T. After that, terminate the word placing a NULL in the pointer of the node holding the T.
  • On the figure, big blocks represent lists, smaller ones represent nodes. We have three lists, each one holding one node. Each node point to a list or NULL.
  • To add the word: ACTUAL: Use the firs char of word: A. A is already on the root list, move to the next list using the list pointer. Take second letter: C. C is already there, use list pointer, move to the third. T is on the list, move to the next. U is NOT in the next list. If there is a NULL, create a list, including the NULL and then create a node and place the U, then the A and lastly the L. Add a NULL at the end.
  • The same way for the word 'ACTUALLY' ont he same tree.
  • To add the word: AND
  • Take the first char, A. A is already in the root list. Use the list pointer from the node holding the A and move to the next list. On second list. Is the char N in the list?. List is not empty, but N is not there. Add a new node (the second for that list) place N in that node. This new node will have a pointer pointing to a new empty list. Now, D. Is D on the list which is addressed from the node holding the N?. No, the list is empty. Create a node and place the D. Add a NULL to the end.
  • On Figure 4 you can see on level 1, the list has two nodes and each node is pointing to a different list, both on level 2.
  • To add the word: BOOK
  • B is not on the root list, which is the first char from new word. Add B to root list. Add the rest of the word. Look at Figure 4.

Multi Linked List
description
The multi-linked list is a console application written in the C programming language. The program is an attempt to learn about memory addresses and memory pointers in detail.
NB: (The program is case sensitive.)

Instruction
  • Assume that we have a list of people with information such as: ID (unsigned int type), Name (String type), Last Name (String type), Email (String type) and Country of Birth (String type). ID should be a positive integer and unique.
Funtionalities
  • MLL *CreateMultiLinkList() that creates a multilinked list. Metadata (Data in the head) should include the number of nodes in the list. The list is a multilinked linear and unidirectional list. MLL is the head struct, which is a user defined type. There should be two pointers on the Head: pID and pLN. pID will address the nodes using the ID field and pLN will address the nodes using the Last Name. An empty list should have NULL in each of those pointers.
  • void InsertNode(MLL *myList, unsigned int ID, char Name[], char LastName[], char Email[], char Country[]) that inserts a node to the list.
  • void PrintByID(MLL *myList) that prints the list, traversing it using pID. Printing will not retrieve the nodes.
  • void PrintByLN(MLL *myList) that prints the list, traversing it using pLN. Printing will not retrieve the nodes.
  • Node *RetrieveNodeByID(MLL *myList, unsigned int ID) that retrieves the node with such ID. Node is the node Struct. Function should return NULL if there is no node with such ID, otherwise, it should return the memory address of the retrieved Node. Take the Node out of the list. --Do not dispose the node from memory--.
  • Node *RetrieveNodeByLN(MLL *myList, char LastName[]) that retrieves the first node with such Last Name. Node is the node Struct. Function should return NULL if there is no node with such Last Name, otherwise, it should return the memory address of the retrieved Node. Take the Node out of the list. --Do not dispose the node from memory--.
Rules
  • When inserting a node, the pID should be used to place the node sorted in ascending ID order. Meaning that: for a non empty list, the head pointer pID should address the Node with the smallest ID, which in turn will address the next smallest one, and so on, ordered from the smallest to the largest. Node with largest ID should be the last one and address the NULL pointer.
  • When inserting a node, you will use the pLN pointer on the Head to insert the new node in an alphabetical ordered way, using the Last Name field as the order index. pLN will address the node with smallest alphabetical Last Name, and this node will address the next smallest one, and so on, ordered from the smallest to the lasgest. Node with largest alphabetical order Last Name should be the last one and it should address the NULL pointer.
